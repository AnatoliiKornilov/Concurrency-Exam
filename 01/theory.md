# 1. Сокеты. Системные вызовы socket, bind, listen, accept. Реализация синхронного однопоточного TCP-сервера.

## Сокеты: общее понятие
Сокет — программный интерфейс для обмена данными между процессами (обычно по сети).

Сокет идентифицируется парой (IP-адрес, порт). В C/C++ работа с сокетами — через системные вызовы ОС.

## Системные вызовы (POSIX)

### `socket()`
Создаёт новый сокет и возвращает его дескриптор (целое число, похожее на файловый дескриптор).

```c
int socket(int domain, int type, int protocol);
```

### Параметры:

+ domain — семейство протоколов (адресации):

+ AF_INET (или PF_INET) — IPv4.

+ AF_INET6 — IPv6.

+ AF_UNIX — локальные сокеты (файловые домены).

+ type — тип сокета:

+ SOCK_STREAM — надёжная передача с установкой соединения (TCP).

+ SOCK_DGRAM — ненадёжная, без установки соединения (UDP).

+ protocol — конкретный протокол. Обычно 0, тогда система выбирает протокол по умолчанию для заданных domain и type (например, для AF_INET + SOCK_STREAM будет TCP).

Возвращает дескриптор сокета или -1 при ошибке.

### `bind()`
Привязывает сокет к локальному адресу (IP + порт). Нужен серверу, чтобы клиенты знали, куда подключаться.

```c
int bind(int sockfd, const struct sockaddr* addr, socklen_t addrlen);
```

+ sockfd — дескриптор сокета.

+ addr — указатель на структуру с адресом. Для IPv4 используется struct sockaddr_in.

+ addrlen — размер структуры.

#### Пример заполнения sockaddr_in:

```c
struct sockaddr_in addr;
addr.sin_family = AF_INET;                 // IPv4
addr.sin_addr.s_addr = INADDR_ANY;         // слушать на всех интерфейсах
addr.sin_port = htons(8080);               // порт (переводится в сетевой порядок)
```

INADDR_ANY означает «все доступные сетевые интерфейсы» (0.0.0.0).
Если указать конкретный IP, то сокет будет слушать только на нём.

### `listen()`
Переводит сокет в пассивный режим ожидания входящих соединений.

```c
int listen(int sockfd, int backlog);
```

+ backlog — максимальная длина очереди ещё не принятых (accept) соединений. Если очередь заполнена, новые подключения могут отвергаться или задерживаться (зависит от ОС).

После listen() сокет становится серверным (пассивным).

### `accept()`

Извлекает первое ожидающее соединение из очереди и создаёт для него новый сокет.

```c
int accept(int sockfd, struct sockaddr* addr, socklen_t* addrlen);
```
+ sockfd — слушающий сокет (после listen).

+ addr — указатель на структуру, куда запишется адрес клиента (может быть nullptr).

+ addrlen — указатель на размер структуры (input-output).

Возвращает дескриптор нового сокета, связанного с конкретным клиентом.
Если очередь пуста, accept() блокируется до прихода нового клиента.

## Синхронный однопоточный TCP-сервер
### Принцип работы:
1. Создаём сокет (socket).

2. Привязываем (bind) к порту и адресу.

3. Начинаем слушать (listen).

4. Входим в бесконечный цикл:

5. Вызываем accept — блокируемся, пока не подключится клиент.

6. Получаем дескриптор клиентского сокета.

7. Читаем данные из этого сокета (read — тоже блокируется, пока клиент не пришлёт данные).

8. Отправляем ответ (write).

9. Закрываем клиентский сокет.

### Ключевая особенность: блокирующие вызовы
+ accept ждёт нового клиента.

+ read ждёт данные от текущего клиента.

Если клиент подключился, но молчит, сервер повиснет на read для этого клиента. Другие клиенты не смогут подключиться, пока первый не отправит данные или не закроет соединение.

### Проблемы синхронной однопоточной модели
+ Плохая масштабируемость – один медленный клиент блокирует всех.

+ Неэффективное использование ресурсов – процессор простаивает в ожидании ввода-вывода.

+ Неприменимость в реальных сетевых приложениях – практически всегда нужна либо многопоточность, либо асинхронный/неблокирующий ввод-вывод.

### Преимущества (только учебные)
+ Простота понимания и реализации.

+ Нет накладных расходов на синхронизацию (мьютексы, очереди).

### Важные вспомогательные функции и детали
+ htons(), htonl(), ntohs(), ntohl(). Порты и IP-адреса в сетевых протоколах передаются в сетевом порядке байт (big-endian). На x86 используется little-endian.
htons (Host TO Network Short) преобразует uint16_t (порт) в big-endian.
htonl — для 32-битных адресов.

+ SO_REUSEADDR - Опция сокета, позволяющая переиспользовать локальный адрес. Когда TCP-сервер завершается (нормально или аварийно), его порт может остаться в состоянии TIME_WAIT на несколько минут (обычно 2–4 минуты). В этом состоянии порт занят ядром и любой новый вызов bind() к тому же порту вернёт ошибку Address already in use. Если вы перезапускаете сервер сразу после остановки, он не сможет запуститься — придётся ждать таймаута или менять порт. Когда вы включаете эту опцию, ядро разрешает новому сокету привязаться к адресу, который находится в TIME_WAIT (при условии, что адрес не используется активным соединением). Это критично для серверов, которые перезапускаются часто. Устанавливается через setsockopt:

```c
int opt = 1;
setsockopt(server_fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));
```

+ Закрытие сокета: close(). Освобождает дескриптор. Для TCP закрытие отправляет FIN.
