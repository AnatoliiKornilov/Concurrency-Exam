# 4. Условные переменные и семафоры

## Проблема: как потоку дождаться выполнения условия без busy waiting

Когда один поток должен подождать, пока другой поток изменит общее состояние (например, появятся данные в очереди), наивный подход – цикл с проверкой флага и `sleep()`. Это неэффективно: тратит процессор или вводит ненужные задержки.

**Решение:** условная переменная (`condition_variable`) позволяет потоку заблокироваться до тех пор, пока другой поток не уведомит его о наступлении события.

## `std::condition_variable`

### Основные методы

- `wait(unique_lock<mutex>& lock)` – атомарно:
  1. отпускает мьютекс,
  2. блокирует поток,
  3. после пробуждения снова захватывает мьютекс.
- `wait(lock, predicate)` – эквивалентно `while(!predicate()) wait(lock);` (защита от spurious wakeups).
- `notify_one()` – пробуждает один из ожидающих потоков (если есть).
- `notify_all()` – пробуждает все ожидающие потоки.

### Требования

- Должен использоваться вместе с `std::unique_lock<std::mutex>` (не с `lock_guard`).
- Всегда должен быть **предикат** (условие), проверяемое после пробуждения, чтобы отличить ложное пробуждение от реального события.

### Типичный паттерн: producer-consumer

**Producer**:
```c
{
    std::lock_guard lock(mtx);
    queue.push(data);
}
cv.notify_one();
```

**Consumer**:
```c
std::unique_lock lock(mtx);
cv.wait(lock, [&]{ return !queue.empty(); });
auto data = queue.pop();
lock.unlock();
// обработка data
```

### Spurious wakeups (ложные пробуждения)
ОС может разбудить поток даже без вызова `notify`. Поэтому проверка условия обязательна в цикле. Использование `wait`(`lock`, `predicate`) автоматически повторяет проверку.

## Семафоры (`std::counting_semaphore`)
+ Семафор – примитив синхронизации, позволяющий ограничить количество потоков, одновременно обращающихся к ресурсу. Считает количество доступных «единиц» ресурса.

`std::counting_semaphore<LeastMaxValue>` (C++20). `LeastMaxValue` – максимальное значение счётчика (компиляция).

`release(n = 1)` – увеличить счётчик на n, разбудить ожидающие потоки.

`acquire()` – уменьшить счётчик; если 0 – заблокироваться.

`try_acquire()` – неблокирующая попытка.

`try_acquire_for() / try_acquire_until()` – с таймаутом.

### Отличие от мьютекса
+ Мьютекс – бинарный (0 или 1) и имеет владельца (только захвативший может освободить).

+ Семафор – может иметь произвольное начальное значение, не имеет владельца (любой поток может вызвать release).
