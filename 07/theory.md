# 7. Системный вызов `futex`. Реализация `std::mutex` через `futex`. Идея реализации `futex` в ядре ОС.

## Проблема традиционных мьютексов

Раньше мьютексы реализовывались целиком в ядре: каждый `lock` и `unlock` требовали системного вызова, даже если конкуренции не было. Это дорого (переключение в ядро).

**Идея futex:** большую часть времени мьютекс работает в userspace (быстро), и только при реальной конкуренции происходит системный вызов.

## Что такое futex

`futex` (Fast Userspace Mutex) – системный вызов Linux, который позволяет потоку **заснуть** и **проснуться** на основе значения переменной в userspace.

Основные операции:
- `FUTEX_WAIT` – проверить, что значение переменной равно ожидаемому, и если да – усыпить поток.
- `FUTEX_WAKE` – разбудить один или несколько потоков, ожидающих на данном адресе.

Синтаксис (упрощённо):
```c
int futex(int* uaddr, int op, int val, const struct timespec* timeout, int* uaddr2, int val3);
```

#include <linux/futex.h>
#include <sys/syscall.h>

``` c
// Заблокировать поток, если *uaddr == val
syscall(SYS_futex, uaddr, FUTEX_WAIT, val, NULL, NULL, 0);

// Пробудить до n потоков, ожидающих на uaddr
syscall(SYS_futex, uaddr, FUTEX_WAKE, n, NULL, NULL, 0);
```

## Идея реализации мьютекса через `futex`
+ Храним переменную state в userspace:

+ 0 – мьютекс свободен.

+ 1 – захвачен, нет ожидающих.

+ 2 – захвачен, есть ожидающие (или просто >1).

### Алгоритм `lock()`:
+ Попробовать атомарно изменить state с 0 на 1 (CAS).

+ Если удалось – мьютекс захвачен (быстрый путь).

+ Если нет – увеличиваем счётчик ожидающих (или ставим state=2) и вызываем futex(FUTEX_WAIT) с ожиданием, что state == 1 (или 2). Поток засыпает.

### Алгоритм `unlock()`:
+ Атомарно уменьшить state на 1.

+ Если state стал 0 – никого не будим.

+ Иначе (было 1 → стало 0) или было 2 → стало 1 – вызываем futex(FUTEX_WAKE) для пробуждения одного ожидающего потока.

## Реализация `futex` в ядре
+ Ядро поддерживает для каждого адреса futex очередь ожидающих потоков.

### `futex_wait(uaddr, val)`:

+ Проверяет, что `*uaddr == val` (под мьютексом ядра).
+ Если равно – добавляет текущий поток в очередь ожидания на этом адресе и переключает контекст (поток засыпает).
+ Если не равно – возвращает ошибку (`EAGAIN`), и пользовательский код должен повторить попытку.
### `futex_wake(uaddr, count)`:

+ Находит очередь ожидающих на `uaddr`.
+ Будит не более `count` потоков (удаляет их из очереди и ставит в `run queue`).
+ Ядро не знает, что именно означает значение val – это логика пользовательского кода.

## Преимущества `futex`:

+ При отсутствии конкуренции – ноль системных вызовов (работа в userspace).

+ При конкуренции – поток засыпает, не тратя CPU в спинлоке.

+ Подходит для любых примитивов: мьютексов, условных переменных, семафоров, barrier.

## Недостатки
+ Сложнее в реализации.

+ Зависимость от ОС Linux (в Windows – WaitOnAddress, в macOS – ulock).

+ Требует аккуратной работы с `memory order`.

## Альтернативы
+ `pthread_mutex` на Linux тоже использует `futex` (через `lll_lock`).

+ `std::mutex` в GCC/clang реализован через `pthread_mutex`, который на Linux – `futex`.
