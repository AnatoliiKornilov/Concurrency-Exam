# 5. Thread pool с блокирующей очередью

## Проблема: создание потока на каждый запрос – дорого

В многопоточном сервере (пункт 2) мы создавали новый поток на каждое клиентское соединение. Это плохо при большом количестве клиентов:
- Создание потока – дорогая операция (выделение стека, системные вызовы).
- Потоки потребляют память (обычно 1–8 МБ на стек).
- При тысячах подключений система может исчерпать ресурсы.

**Решение**: пул потоков (thread pool).

## Идея thread pool

- При старте создаётся фиксированное количество потоков (например, по числу ядер процессора или чуть больше).
- Поступающие задачи (клиентские сокеты, вычисления) складываются в общую очередь.
- Потоки из пула разбирают эту очередь: каждый берёт задачу, выполняет, затем берёт следующую.

## Преимущества
- **Ограничение параллелизма** – не больше потоков, чем в пуле.
- **Переиспользование потоков** – без накладных расходов на создание/уничтожение.
- **Плавная нагрузка** – очередь сглаживает пики.
- **Масштабируемость** – можно настроить размер пула под систему.

## Блокирующая очередь (blocking queue)

Очередь задач должна быть потокобезопасной. Если очередь пуста, рабочие потоки должны **блокироваться** (ждать) до появления новой задачи. Для этого используется связка `std::mutex` + `std::condition_variable`.

### Требования к блокирующей очереди
- `push(task)` – добавляет задачу в конец, уведомляет один ожидающий поток (`notify_one`).
- `pop()` – извлекает задачу из начала; если очередь пуста – блокируется, пока задача не появится.
- Возможность остановки (shutdown) – чтобы потоки могли корректно завершиться.

## Классическая реализация thread pool

### Компоненты
1. **Очередь задач** – `std::queue` или `std::deque` под мьютексом.
2. **Condition variable** – для блокировки потоков при пустой очереди.
3. **Флаг остановки** – `stop` (обычно `bool` или `std::atomic<bool>`).
4. **Вектор потоков** – `std::vector<std::thread>`.

### Алгоритм работы рабочего потока (worker loop)
```cpp
while (true) {
    std::function<void()> task;
    {
        std::unique_lock<std::mutex> lock(mtx);
        cv.wait(lock, [this]{ return !tasks.empty() || stop; });
        if (stop && tasks.empty())
            break;
        task = std::move(tasks.front());
        tasks.pop();
    }
    task();
}
```
